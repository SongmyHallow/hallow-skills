# 游戏行业舆情分析

## 概述

适用于**任意游戏**的端到端舆情处理：多平台数据收集 → 分类整理 → 分析洞察 → HTML 可视化报告。

> 详细平台检索策略见 `references/platform-strategies.md`，包含每个平台的 JS 提取代码、CDP 操作方法、已知陷阱。

## 工作流程

### 第一步：需求确认

在收集前与用户确认以下参数，未明确的主动询问：

| 参数 | 说明 |
|------|------|
| 分析模式 | 快速扫描 / 专题分析 / 周期性报告 |
| 目标平台 | 抖音、B站、小红书、贴吧、微博（五大主平台）；特殊情况可加 NGA/TapTap/应用商店 |
| 时间范围 | 起止日期 |
| 关键词 | 核心搜索词和辅助词 |
| 样本量 | 每平台采集量（默认 20-50 条） |
| 游戏名称 | 目标游戏全称，决定报告命名和输出路径 |

### 第二步：数据收集

#### 2.1 前置准备

在访问每个平台前，**必须**读取 web-access 的对应站点经验文件：

| 平台 | 站点经验 | 有无 |
|------|---------|------|
| 抖音 | `skills/web-access/references/site-patterns/douyin.com.md` | ✅ |
| B站 | `skills/web-access/references/site-patterns/bilibili.com.md` | ✅ |
| 小红书 | `skills/web-access/references/site-patterns/xiaohongshu.com.md` | ✅ |
| 贴吧 | `skills/web-access/references/site-patterns/tieba.baidu.com.md` | ✅ |
| 微博 | `skills/web-access/references/site-patterns/weibo.com.md` | ✅ |

**读取时机**：确定目标平台后批量并行读取，不要等到访问时才读。

#### 2.2 工具选择：WebSearch vs CDP

**原则：能用 WebSearch 快速摸底的不上来就开 CDP。但小红书和抖音评论必须 CDP。**

| 平台 | 快速扫描 | 专题/周期报告 |
|------|---------|-------------|
| 贴吧 | WebSearch（标题即态度） | WebSearch + CDP 帖子页 |
| B站 | WebSearch 概览 | WebSearch + CDP fetch 评论 API |
| 抖音 | CDP 搜索页标题+点赞 | CDP 搜索页 + 详情页评论 |
| 小红书 | CDP 搜索页 innerText | CDP 搜索页 + 详情页 |
| 微博 | WebSearch 尝试（不行就跳过） | CDP 搜索页 |

**WebSearch 搜索词构造原则：**
- 第一轮：`"{产品名} {核心话题}"` 宽泛搜索，看返回量和来源分布
- 第二轮：对声量最高的平台加 `site:` 限定精准搜索，如 `永劫无间 周年庆 site:tieba.baidu.com`
- 如果第一轮某平台无结果，不要反复尝试——直接标记声量低或切换到 CDP

#### 2.3 并行执行策略

**快速扫描模式**：同时启动 WebSearch 多平台搜索 + CDP 打开抖音/B站搜索页。利用并行等待时间一次性拿到所有平台概况。

**专题/周期模式**：先并行 WebSearch 获取全貌，再根据声量选择重点平台做 CDP 深度抓取。

#### 2.4 平台优先级（按话题选平台）

| 话题类型 | 优先平台（快扫只选 3 个） |
|----------|-------------------------|
| 版本更新/福利争议 | 贴吧 + B站 + 抖音 |
| 新英雄/新皮肤 | B站 + 抖音 + 小红书 |
| Bug/技术问题 | 贴吧 + B站 |
| 破圈事件/公关危机 | 微博 + 抖音 + B站 |
| 付费/商业化 | 贴吧 + 小红书 + B站 |

#### 2.5 收集规范

- 优先采集高互动量内容（高赞、高回复、高转发）
- 每条记录附：平台、发布时间、互动量、作者、URL
- 仅收集**公开可见**内容；严格控制在合理频率
- 每平台采集完毕后立即整理，避免数据积压
- 搜索无结果时标记"该平台声量低"，不反复重试

### 第三步：分类整理

对采集到的每条舆情进行标注：

**情感标签：** 😊 正面 / 😐 中性 / 😞 负面

**主题标签：** 按内容归类。以下为基础分类，每次调研前与用户确认当前最关心的维度并相应调整：
- 英雄/角色平衡、付费/商业化、Bug/技术问题、运营活动、匹配/排位体验、UI/UX、剧情/世界观、社区氛围、客服反馈、竞品相关

**热度标签：** 🔥 高（>1000互动）/ 🔶 中（100-1000）/ 〰️ 低（<100）

**紧急度标签：** 🚨 需立即关注 / ⚠️ 常规关注 / ℹ️ 仅供参考

### 第四步：分析洞察

按分析模式差异化处理：

**快速扫描模式**（版本更新当天，舆论未完全发酵阶段）：
1. **定位**：版本更新/活动上线当天，快速判断初期舆情风向，此时舆论碎片化、尚未形成共识
2. **关注重点**：是否有重大事故（无法登录、严重bug、付费翻车）；玩家第一印象（正面多还是负面多）；有没有快速扩散的风险点
3. 舆情基调判断（正面/负面/混合为主）
4. TOP 3-5 热点话题及占比
5. 紧急风险点（如有）
6. 一句话核心结论 + 是否建议持续关注
7. 直接进入报告生成，轻量化处理

**专题分析模式**（围绕特定主题深挖）：
1. 多维拆解：按情感 × 话题交叉分析
2. 玩家核心诉求提炼（不只看表面抱怨，追问背后需求）
3. 代表性言论摘录（正面/负面各选 3-5 条）
4. 与历史对比（如有历史数据）
5. 具体改进建议（分紧急度和可行性）

**周期性报告模式**（周报/月报/版本报告）：
1. 周期舆情总体趋势
2. 各渠道声量分布与变化
3. 情感走势变化（与上期对比）
4. 热点话题排行榜（TOP 10）
5. 本期新增问题 vs 持续问题 vs 已解决问题
6. 下期关注点预测

**跨平台对比分析**（所有模式都做）：
- 同一话题在不同平台的情感差异往往比总体数字更有价值
- 贴吧偏负面/硬核、抖音偏正面/轻度、B站偏内容驱动、小红书偏消费审美——这是正常偏差
- 当某个平台的情感出现异常反转（如贴吧一面倒夸、抖音一面倒骂），才是值得警惕的信号

### 第五步：HTML 报告生成

#### 输出路径

报告默认输出到当前工作目录。如用户有指定路径（如项目文档目录），优先使用用户指定路径。

命名格式：`{游戏名}-舆情{模式}-{主题}-{yyyyMMdd}.html`

示例：
- `永劫无间-舆情快速扫描-五周年预热-20250625.html`
- `D90-舆情专题-新英雄魏轻上线反馈-20250625.html`
- `L22-舆情周期-六月周报-20250625.html`

> 在 D90/L22 项目中使用时，按 CLAUDE.md 规则输出到 `E:\{项目}_文档草案\UX-数据分析\`。

#### 报告模板结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>舆情报告标题</title>
  <script src="https://cdn.jsdelivr.net/npm/echarts@5.5.0/dist/echarts.min.js"></script>
  <style>/* 报告样式 */</style>
</head>
<body>
  <!-- 1. 报告头部：标题、时间、模式、覆盖平台 -->
  <!-- 2. 核心发现卡片（3-5 条，醒目展示，使用颜色区分信息/警告/危险级别） -->
  <!-- 3. 舆情仪表盘：情感饼图 + 平台声量柱状图 + 热点排行 -->
  <!-- 4. 分平台详情：每个平台的收集概况和代表性言论（附引用块） -->
  <!-- 5. 话题深度分析：各主题的舆情分布和关键洞察 -->
  <!-- 6. 风险预警：需关注的问题、触发条件、建议行动 -->
  <!-- 7. 附录：数据来源、采集方法、样本量说明 -->
  <script>/* ECharts 图表初始化 */</script>
</body>
</html>
```

#### 图表要求

使用 ECharts CDN 渲染交互式图表，至少包含：
- **情感分布饼图**：三色（绿=正面、红=负面、灰=中性）
- **平台声量柱状图**：各平台采集量和情感构成（堆叠柱状图）
- **话题热度横向柱状图**：TOP 10 话题及提及次数，颜色区分话题性质（红=争议/绿=正面/蓝=内容）

色彩方案统一使用：
- 正面：`#52c41a`
- 负面：`#f5222d`
- 中性：`#8c8c8c`
- 主色调：`#1890ff`
- 警告：`#faad14`

#### 样式规范

- 卡片式布局，白色背景 + 浅灰底色（`#f5f6fa`）
- 代表性言论使用引用块样式，左侧色条区分正负面，标注平台和点赞数
- 紧急风险使用红色边框高亮（`border: 2px solid #f5222d` + 浅红背景）
- 核心发现使用彩色左边框卡片（蓝=信息/黄=警告/红=危险）
- 移动端响应式适配（图表行列在窄屏下堆叠）

## 快速扫描模式精简版

快速扫描的核心是**快**——舆论尚未定型时给出初步判断，而非精确结论。

1. 选定 2-3 个最相关平台（按第二步的平台优先级表选）
2. 并行 WebSearch + CDP 搜索页，一次性拿到概况
3. 提取标题和点赞数即可，不进详情页和评论区
4. 记录关键印象和 3-5 条代表性言论
5. 生成精简一页报告：总体判断 + 热点 TOP 5 + 是否有红色警报 + 建议是否持续跟踪
6. 图表可简化为纯 CSS 柱状图，不强制 ECharts
7. **目标时长：15-30 分钟**

## 注意事项

**访问合规：**
- 仅收集公开可见内容，不尝试绕过登录墙
- 控制访问频率，每个平台间隔 3-5 秒
- 小红书短时间内大量打开 tab 会触发验证码，控制在 10 个 tab 以内
- 报告中摘录原文不超过 200 字/条，标注出处

**数据真实性：**
- 注意识别水军/刷评迹象（大量同质化内容、新号集中发声）
- 如发现异常模式，在报告中单独标注
- 抖音/小红书存在大量营销号重复内容，注意去重

**时效性：**
- 快速扫描模式：从开始到报告完成控制在 30 分钟内
- 专题分析/周期性报告：按数据量预计时间并提前告知用户

**平台差异化认知：**
- 贴吧偏负面不等于舆情全面负面，这是平台属性
- 抖音偏正面不等于舆情全面正面，这也是平台属性
- 关键在于：看每个平台的情绪是否出现了**偏离其正常基线**的反转
